# SQL-Based Restaurant Sales Analysis

## Project Overview
This project demonstrates the transition from flat-file data to a relational database. I migrated a restaurant dataset into a SQL environment to perform complex business aggregations.

## Tech Stack
- **Database:** SQLite3
- **Analysis:** Python (Pandas)
- **Visuals:** Seaborn

## Key Technical Steps
- **ETL Process:** Successfully migrated CSV data into a **Relational Schema** using `sqlite3`.
- **SQL Aggregations:** Wrote complex queries using `GROUP BY`, `SUM`, and `AVG` to determine peak revenue days.
- **Reporting:** Bridged the gap between SQL results and Python visualization to create a Daily Revenue Report.

## 💡 Insight
The analysis identified Saturday as the peak revenue day with the highest average tips, providing actionable data for staff scheduling.
